0.2 PRELIMINARIES GOAL

At ticked, we aim to create a professional relationship between the user and the Virtual Assistant (VA). The user creates a task and chooses to assign it to the Virtual Assisant who will carry out each of those created tasks.

PROPOSED LOGO

The Ticked logo is formed by a unique shape with curved edges, which aims to bring a neat design. The warm, orange tone of the logo also makes the logo more approachable and keeps the design simple. The logotype of Ticked gives the user a feeling of "accomplished" when seen.

0.3 RESEARCH

USER INTERVIEW

We conducted 3 interviews with potential users, we asked them questions regarding what they wish to see differently in a To-do list app and how they plan their day. Below are their quotes:

ESE ASOGBO

“As a CEO, I get things mixed up sometimes while trying to prioritize my day. I want to be reminded always.”

KELVIN ADAMS

“I want an app that allows a VA handles all my task for the day.”

AMANDA EZRA

“I would like to see an app that offers VA services inclusive of their skills.”

TARGET AUDIENCE

Of course anyone can use the Ticked app. However, we went for specifications due to monitization of the product.

Business Class People who will require the VA services are the entrepreneurs who are always busy with loads of daily schedules and have little or no more to complete their task for the day.

Successful Youngies

These are mostly millennials or Gen Z’s who own a thriving business and are sometimes occupied with social activities.

0.6 USABILITY TESTING

We sketched out wireframes for the project and began our user testings with people outside our team.

LOW - FI WIREFRAMES + TESTING Here is one key takeaway that we got from our user testing on the low-fi wireframes: Since it is a To-do list app that help users complete a task, there should be a “motivational quote” that encourages the user upon arriving at the homepage before creating a task.
RESULT: The user gets a motivational quote for everytime he wants to create a task. By doing so, the user is motivated to complete a task.
0.8 final design main features
Dashboard
1 Users are able to chat with the Virtual Assistant in case of a task request or task progress.
2 Users are able to create a task new and assign to a VA according to the skills they possess.
3 The user is able to see the task assigned to the VA. At this interface, the user is also able to change a VA.
The “DONE”, “PENDING’’, and ‘’ OVERDUE ’’ which are highlighted above shows the user progress of each task created.
Create Task
1 Users are able to assign task to a VA if required, select date and time of task reminder. The user also has an option to select none if a VA is not required.
VA Prompt
1 The modal appears when a task has been created. As a subscribed user, it progresses to more information about the task reminder and as an unsubscribed user, it gives an opportunity to subscribe to a plan to use the VA feature.
2 Users can also ignore the modal if a VA is not required.
VA Dashboard
1 Here, the VA keeps track of users task progress. The user is able to see when a task is ’’OVERDUE‘’and seeks the attention of the user.
Task Details
1 The VA is able to see full details of the task assigned by the user.
